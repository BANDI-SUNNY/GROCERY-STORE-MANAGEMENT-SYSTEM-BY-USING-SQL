# GROCERY-STORE-MANAGEMENT-SYSTEM-BY-USING-SQL

This project is a SQL-based Grocery Store Management System designed to manage products, suppliers, customers, employees, and sales transactions.

## Objectives

* Analyze customer purchasing behavior
* Identify top-selling products
* Evaluate supplier and employee performance
* Track sales trends
* Generate business insights

## Database Design

The project contains 7 tables:

* Supplier
* Categories
* Products
* Customers
* Employees
* Orders
* Order_Details

## SQL Concepts Used

* Joins
* Aggregate Functions
* GROUP BY
* HAVING
* Subqueries
* CASE Statements
* Foreign Keys
* ER Modeling

## Key Insights

* Top customers contribute significantly to revenue
* Certain suppliers contribute most sales
* Employee performance varies across orders
* Sales patterns help with business decision-making
